# hello-argocd-bootstrap

GitOps bootstrap repository for Argo CD. Implements the **apps-of-apps** pattern with a matrix-based `ApplicationSet` for automatic service discovery.

**Repository:** `https://github.com/geekahmed-yt/hello-argocd-bootstrap.git`

---

## Repository Structure

```text
hello-argocd-bootstrap/
├── bootstrap/
│   ├── root-application.yaml               # One-shot applied manually; reconciles bootstrap/apps
│   └── apps/
│       ├── kustomization.yaml              # Kustomize entry-point with GITHUB_ORG replacement
│       ├── bootstrap-params.env            # Non-secret parameters (GITHUB_ORG)
│       ├── application-argocd-core.yaml         # wave -10
│       ├── application-argocd-projects-rbac.yaml  # wave  -5
│       ├── application-argocd-repos.yaml          # wave  -4
│       └── applicationset-hello-manifests.yaml    # wave   0
└── platform/
    └── argocd/
        ├── core/
        │   ├── argocd-cm.yaml              # Argo CD core ConfigMap
        │   └── cmp/
        │       ├── plugin-cm.yaml          # hello-infra-resolver CMP plugin definition
        │       ├── script-cm.yaml          # Resolver script (reads service-contract.yaml)
        │       └── repo-server-patch.yaml  # Adds CMP sidecar to argocd-repo-server
        ├── projects-rbac/
        │   ├── appproject-platform.yaml    # platform AppProject
        │   └── argocd-rbac-cm.yaml         # RBAC policy ConfigMap
        └── repos/
            └── kustomization.yaml          # Intentionally empty — repo creds are manual
```

## Runbook

### Step 1 — Create the GitHub token Secret (ApplicationSet discovery)

The ApplicationSet uses this token to scan the GitHub organisation for manifest repos.

```bash
kubectl -n argocd create secret generic argocd-github-token \
  --from-literal=token=<your-github-token>
```

### Step 2 — Create repo credentials (Argo CD sync access)

Provides Argo CD with credentials to clone discovered repos. Applied as an org-level credential template so it covers all matching repos automatically.

```bash
kubectl -n argocd create secret generic repo-creds-github-org \
  --from-literal=url=https://github.com/geekahmed-yt \
  --from-literal=type=git \
  --from-literal=username=x-access-token \
  --from-literal=password=<your-github-token>

kubectl -n argocd label secret repo-creds-github-org \
  argocd.argoproj.io/secret-type=repo-creds --overwrite
```

### Step 3 — Apply the root Application (once only)

This is the only manual `kubectl apply` ever needed. From this point Argo CD manages itself.

```bash
kubectl apply -f bootstrap/root-application.yaml
```

### Step 4 — Verify bootstrap wave ordering

Argo CD reconciles `bootstrap/apps` and creates child apps in sync-wave order:

```bash
kubectl -n argocd get applications -w
```

Expected order and final state:

| Wave | Application | Manages |
|------|-------------|---------|
| -10 | `argocd-core` | `argocd-cm`, CMP sidecar on `argocd-repo-server` |
| -5 | `argocd-projects-rbac` | `platform` AppProject, RBAC ConfigMap |
| -4 | `argocd-repos` | (empty — repo creds are manual) |
| 0 | `hello-manifests` ApplicationSet | All discovered `hello-*-manifests` repos |

### Step 5 — Verify ApplicationSet repo discovery

```bash
# List all Applications generated by the ApplicationSet
kubectl -n argocd get applications -l app.kubernetes.io/managed-by=argocd-applicationset

# Inspect which repos were discovered
kubectl -n argocd describe applicationset hello-manifests
```

### Step 6 — Verify the CMP sidecar is running

The `hello-infra-resolver` CMP reads `service-contract.yaml` from manifest repos and injects resolved infra dependency components.

```bash
# Confirm the sidecar is present
kubectl -n argocd get pod -l app.kubernetes.io/name=argocd-repo-server \
  -o jsonpath='{.items[0].spec.containers[*].name}'
# Expected output includes: hello-infra-resolver

# Tail CMP logs
kubectl -n argocd logs -l app.kubernetes.io/name=argocd-repo-server \
  -c hello-infra-resolver -f
```

## Steady-State Operations

### Adding a new manifest repo

Push a repo to the `geekahmed-yt` organisation with a name matching `hello*manifests`. The ApplicationSet discovers it on the next reconciliation cycle and creates an Argo CD Application automatically.

### Updating Argo CD platform configuration

Edit files under `platform/argocd/` and push to `main`. The relevant child app (`argocd-core`, `argocd-projects-rbac`) self-heals within the reconciliation window.

### Rotating the GitHub token

Secrets are never in Git. Update in-cluster only:

```bash
kubectl -n argocd create secret generic argocd-github-token \
  --from-literal=token=<new-token> \
  --dry-run=client -o yaml | kubectl apply -f -

kubectl -n argocd create secret generic repo-creds-github-org \
  --from-literal=url=https://github.com/geekahmed-yt \
  --from-literal=type=git \
  --from-literal=username=x-access-token \
  --from-literal=password=<new-token> \
  --dry-run=client -o yaml | kubectl apply -f -
```

### Updating the Argo CD / CMP image version

Edit the image tag in `platform/argocd/core/cmp/repo-server-patch.yaml` and push to `main`.

## Architecture Decisions

| Decision | Rationale |
|---|---|
| Root app uses `default` project | Avoids first-sync dependency on `platform` project (created at wave -5) |
| Bootstrap child apps use `default` project | Same — `platform` project does not exist until wave -5 completes |
| Generated apps use `platform` project | Scoped permissions once the project exists |
| Repo creds managed manually | Credentials must never be committed to Git |
| `applicationsSync: create-update` | Prevents ApplicationSet deleting apps on repo rename |
| `preserveResourcesOnDeletion: true` | Protects cluster resources if an Application is removed |
| `goTemplateOptions: missingkey=error` | Fails fast on template variable errors |
| CMP `hello-infra-resolver` | Reads `service-contract.yaml` and injects infra components without touching app repos |
| `GITHUB_ORG` via kustomize replacement | Keeps org name out of hardcoded manifest values |
